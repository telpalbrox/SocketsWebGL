WebSockets + WebGL

Introducción

Vamos a hablar sobre WebSockets y WebGL. Nuestro proyecto ha consistido en, a partir de un pequeño prototipo hecho en WebGL, implementar mucha funcionalidad extra y sockets para que este prototipo se pueda retransmitir en tiempo real a muchos navegadores a la vez.

Sockets

Empezaré hablando sobre sockets y cómo los hemos implementado en el navegador y servidor con el lenguaje JavaScript.

¿Qué es un socket?

Según Wikipedia:

Socket designa un concepto abstracto por el cual dos programas (posiblemente situados en computadoras distintas) pueden intercambiar cualquier flujo de datos, generalmente de manera fiable y ordenada.

¿Qué quiere decir esto?

Pues básicamente que un socket es como un túnel que enlaza varias instancias de tu programa con un servidor. Todo lo que introduzcas en el socket puede llegar a todos los clientes. El socket sobre TCP garantiza que llegará bien y de forma ordenada.

¿Por qué usar sockets en el navegador?

Los sockets son necesarios principalmente cuando se necesita una comunicación en tiempo real y de baja latencia.

Se suelen usar para juegos multijugador, chats, actualizaciones en tiempo real, etcétera.

¿Cómo se usan los sockets en el navegador?

En el navegador los sockets están implementados a través de WebSockets. Este tipo de sockets están implementados sobre un único socket TCP que puede enviar y recibir datos de forma simultánea.

Para usarlos en el navegador, HTML 5 dispone de una API que incluye los WebSockets y que se puede usar a través de JavaScript.

En nuestro caso vamos a usar una librería llamada Socket.io que funciona sobre la API de HTML 5 en JavaScript y que simplifica considerablemente la tarea.

¿Cómo se implementan los WebSockets de lado del servidor?

El socket del navegador necesita conectarse a un servidor para ir enviando y recibiendo los distintos mensajes. Este servidor puede ser el mismo que le proporciona la página web al navegador.

En el lado de servidor hay muchas más opciones. Hay librerías disponibles para los principales lenguajes de programación: JavaScript, Java, .NET, PHP, Python, Ruby, etcétera.

En nuestro caso, como comentábamos anteriormente, se ha elegido la librería Socket.io para JavaScript que está implementada sobre Node.js.

¿Quienes usan Socket.io?

Según su página oficial la versión web de Microsfot Office y el servicio de organicación Trello usan socket.io a parte de otros muchos servicios menos conocidos como Yammer o Zendesk.

Gráficos

En esta parte de la presentación vamos a explicar que tecnologías usa la parte gráfica del prototipo.

WebGL

Para poder dibujar gráficos en 3D en navegadores web es nesario usar WebGL.

WebGL es una especificación estándar para mostrar gráficos 3D en navegadores web accelerados por hardware (GPU).

Técnicamente es una API para JavaScript que permite usar la implementación nativa de OpenGL ES 2.0, y en algunos navegadores hasta OpenGL ES 3.0.

Problema WebGL

El principal problema de WebGL es que es una tecnología diseñada para trabajar directamente con la GPU, por lo que es muy difícil trabajar directamente sobre WebGL.

Three.js

Como es muy difícil trabajr directamente sobre WebGL es necesario usar una librería que nos abstraiga de WebGL.

En nuestro caso hemos elegido Three.js por ser una de las más populares.
